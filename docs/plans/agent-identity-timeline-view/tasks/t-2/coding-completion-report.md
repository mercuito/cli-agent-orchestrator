## Hiccups

- Initial focused test command attempts used `web/src/...` paths while running
  from `web/`, so Vitest found no files. Re-running with `src/test/...`
  confirmed the intended red failures.
- The first implementation-plan review requested explicit `B-7` coverage; the
  plan and Coding Test Contract were revised and then approved.
- Final code review identified that related-event cache state could still be
  repopulated by stale in-flight requests. The implementation now keys related
  cache, error, and loading state by selected identity plus event ID.
- Final test review identified that public-boundary proof rendered only the
  child identity timeline component. A smoke test now renders `AgentPanel` and
  proves the identity timeline panel is wired through the Agents boundary,
  grouped separately from terminal deep-link tests.
- Component tests initially used unique-text queries for repeated event names
  and headings. The tests were adjusted to assert the intended behavior with
  `getAllByText` and role-scoped heading queries.

## Optimization Opportunities

- Future dashboard work could split existing `AgentPanel` session/terminal
  workflows into smaller components. That was kept out of scope because this
  task only needed a focused identity timeline addition.
- If more event surfaces are added, shared frontend formatters for event labels
  and timestamps may become worthwhile. For this slice, local helpers kept the
  change smaller.
