# Local Planning Workspace — Completion Report

Single record of implementation progress and review findings across the
tasks under [tasks/](tasks/).

## Conventions

For each task, add a section `## Task NN: <name>` with subsections as
work progresses:

- **Implementation Summary** — what landed, file diff summary.
- **Verification Evidence** — commands run plus outcomes (pytest module
  output, manual flow notes).
- **Criteria Evaluation** — applicable entries from the `docs/criteria`
  catalog evaluated against this task's diff. List the criterion id,
  whether it applied, and a brief evaluation note.
- **Review Revisions** — one subsection per review finding that required
  an implementation change. Record what the reviewer found, why it was
  accepted as valid, how it was fixed, and what evidence verifies the
  fix.
- **Clean Review Passes** — list each successive clean review pass with
  reviewer identity and timestamp.

A task is complete only after two successive clean review passes are
recorded here for that task. The plan is complete when all tasks are
complete.
