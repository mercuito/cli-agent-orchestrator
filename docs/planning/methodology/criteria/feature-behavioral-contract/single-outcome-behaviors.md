# Behaviors Have One Outcome

Each behavior describes one outcome. If one behavior would need two
different tasks to finish it, split it.

## Illustrations

**Bad — two outcomes in one behavior.** `B-2` says a Linear mention row
shows issue details and has an external issue reference.
**Good:** `B-2` says the Linear mention row shows issue details. `B-7`
says the issue reference opens the Linear issue.

**Bad — presenter plus navigation.** `B-3` says a runtime delivery row
shows delivery details and has an internal terminal reference.
**Good:** `B-3` says the runtime delivery row shows delivery details.
`B-8` says the terminal reference focuses the receiving terminal.
